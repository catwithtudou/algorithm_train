# Algorithm Train

> Rust & Go algorithm practice, mostly around LeetCode daily problems and small data-structure exercises.  
> 用 Rust / Go 长期刷题、复盘算法和保持工程手感的训练仓库。

这个仓库不是一次性整理出来的题解合集，而是一个持续更新的训练记录：截至 2026-05-11，仓库已经有 800+ Go 题解、700+ Rust 练习文件和 876 次提交。它更像一条长期学习轨迹，用来记录我如何把算法题、语言特性和日常编码节奏串起来。

## What is inside

| Area | Path | Notes |
| --- | --- | --- |

## Reading guide

- 想快速看 Go 题解：从 [`go/leetcode`](./go/leetcode) 按题号进入。
- 想看 Rust 练习：从 [`rust`](./rust) 目录按模块浏览。
- 想看更新节奏：可以直接看 commit history，它比 README 更能反映这个仓库的真实用途。

## Why keep this public

- **保持长期训练节奏**：算法训练对我来说不是短期冲刺，而是持续维护基本功。
- **对比 Go / Rust 表达**：同类问题用不同语言写一遍，可以暴露语言模型、数据结构和错误处理习惯的差异。
- **留下可追踪的学习痕迹**：不是每个文件都完美，但真实的迭代过程本身也有价值。
